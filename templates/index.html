BookReconciler Service

Configuration and Tools

Welcome! The service is running at:

${appBase}$api/v1/reconcile copy

Title Matching Behavior:

Single Match Mode

  • Matches to the best Manifestation it can find
  • Pass Title and Author to get best match
  • If passed Publication Year it will try to match the desired edition

Cluster Match Mode

  • Will build a cluster of matching Manifestations which represent the work as a whole
  • Pass Title and Author to get best match
  • Best mode to deal with works that have many editions
  • Best mode to gather as many identifiers as possible

Extend Data Behavior:

Join Mode

123456789 | 987654321 | 192837465 | 564738291
  • When extending data (adding ISBN, LCCN, OCLC, etc..) values will be in the same field seperated by a pipe "|" character

New Row Mode

123456789 987654321 192837465 564738291
  • When extending data (adding ISBN, LCCN, OCLC, etc..) values will be in a new row for each value

Remove Subtitle from Titles:

Keep Subtitles

  • Leave title strings as they are in the data

Remove Subtitles

  • Attempt to remove subtitle from any title strings

Instructions

  1. Click "Build Database" to start the automated process
  2. The system will automatically find and download the latest HathiTrust data file
  3. You should have 5GB of free disk space available
  4. The file will be processed and imported into the database
  5. The process may take several minutes depending on file size and internet speed
  6. Temporary files will be automatically cleaned up after completion
  7. Once complete, the database will be ready for reconciliation

Note:

The database file will be created at data/hathi/hathitrust.db

No manual file downloads or placement required - everything is automated!
